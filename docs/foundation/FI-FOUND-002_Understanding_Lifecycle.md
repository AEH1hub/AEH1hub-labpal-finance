# FI-FOUND-002 — Understanding Lifecycle

**Status:** CANDIDATE — FOUNDATION REVIEW REQUIRED

## Lifecycle

1. Need
2. Question
3. Reality
4. Observation
5. Evidence
6. Validation
7. Preservation
8. Connection
9. Context
10. Interpretation
11. Challenge
12. Revision
13. Current Understanding
14. Review
15. History

## Current Understanding requirements

Every Current Understanding must identify:

- scope;
- supporting evidence;
- challenging evidence;
- material unknowns;
- unresolved contradictions;
- source provenance;
- review state;
- previous understanding;
- changes since the previous understanding;
- next evidence needed.

## Understanding states

- `UNFORMED`
- `EVIDENCE_GATHERING`
- `WORKING_INTERPRETATION`
- `CHALLENGED`
- `REVISION_PROPOSED`
- `REVIEWED`
- `VERIFIED_WITHIN_SCOPE`

Verification is always scoped.

No state may be upgraded solely because:

- an AI generated it;
- a human selected it;
- multiple sources agreed;
- an outcome was profitable;
- it sounds plausible.

## Explanation requirement

LabPal must answer:

> Why do I currently understand this?

It should also answer:

- What made this understanding possible?
- What method was used?
- What remains uncertain?
- What could falsify or revise it?
- Should the same method be used again?
- Was another method more effective?

# OPS-PLAY-008 — Release Review Playbook

**Status:** CANDIDATE — OPERATIONAL REVIEW REQUIRED

## Purpose

Determine whether a capability, milestone, runtime package, or institutional standard has earned release within an explicit scope.

## Release states

- `NOT_READY`
- `READY_FOR_REVIEW`
- `APPROVED_WITHIN_SCOPE`
- `REJECTED_WITH_REASON`
- `REVISION_REQUIRED`
- `RELEASED`
- `SUPERSEDED`
- `RETIRED`

## Procedure

1. Confirm scope and dependencies.
2. Review supporting and challenging evidence.
3. Review unknowns, failures, and contradictions.
4. Run required regressions.
5. Confirm documentation and runtime agree.
6. Confirm constitutional compliance and authority.
7. Confirm rollback, supersession, or retirement path.
8. Preserve the explanation record.

## Approval criteria

Required evidence exists, tests pass within scope, limitations are explicit, failures remain preserved, authority approves, claims do not exceed demonstrated capability, and reversal is possible where needed.

## Rejection criteria

Evidence missing, claims exceed scope, regressions fail, unknowns are concealed, documentation contradicts runtime, authority is missing, or responsible reversal is impossible.
